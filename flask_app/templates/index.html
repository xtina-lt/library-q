Full Stack Developer

A full stack developer is someone who can develop both client and server applications as well as database creation/manipulation. Full stack is defined as the collection of a series of technologies needed to complete software. Being able to create a prototype and read code in different languages allows for versitility in any project. Learning the basics in many languages enhances a larger picture to be taken from complex problems. This is beneficial for unique solutions with a greater over all control of the product.

Request Response Cycle

Client computers and other internet-connected devices request access to a web page. This can be done by inputing URL, clicking links, submitting forms, or refreshing. Client devices talk to servers that store web pages or applications. Then, the application's servers interpret what information is neccessary, sometimes this is done with routing. Finally the application's server sends the data back to the client. This is said to be the response.

Front-End

Front-end developement focuses on what the client sees and interacts with directly, also known as client-side developement. User interface(UI) access points are where client can interact with designs and determine what each part of the application does. Ensuring dynamic readability between cross-browsers, cross-platforms (operating systems), and cross-devices is pertinent to front-end development.

Examples: Hypertext Markup Language(HTML), Cascading Style Sheets(CSS), and JavaScript(JS) frameworks.

Back-End

Back end development creates the server side application processing algorithms. This deals more with the response portion of the Request-Response Cycle. This is also where connection to database(s) is/are initiated. Routes are made to tell servers what information needs to be processed. The execution phase verifies requests, and completes complex algorithms to manipulates data. Finally this data is sent back as a response. Back-end development acts as a middle man for communication between the client and database.

Examples: Python, Java, Ruby, and C#.

Database

Databese developement deals with Structured Query Language(SQL). SQL is a standardized programming lanuage that is used to manage relational databases and perform various operations within them. In this development we create, manipulate, and save data into databases. Databases are a collection of organized information that can be easily accessesd, managed, and updated. Understanding of Entity Relationship Diagrams(ERD's) is fundamental to organize data and minimize repitition while creating visual planes of how the database looks and behaves. Differentiating Many-to-Many, Many-to-One, and One-to-Many relationships, and seperating for normalization is fundamental. The basics are being able to destinguish the differnt datatypes, naming conventions, and CRUD(Create, Read, Update, Delete) quieries.

Examples: SQL, MySQL, MangoDB.

Object-Oriented Programming(OOP)

Object-Oriented Programming is a process that involves analyzing and designing our application from an object oriented point of view. Everything in the world is an object. Objects have properties and behaviors which can be translated into a language that the computer can understand. We use objects and classes to represent real world objects. Variables and attributes are created for things hat real world objects have. Methods and functions are created to represent real world objects' behaviors. Complex planning and implimentation is neccessary to execute this type of programming.

Procedural Programming

Procedural execution is completed by both the developer and the computer. The computer reads from top to bottom. This is how we write out our code. This is sequential.

Model-View-Controller(M.V.C.) Programming

MVC programming is OOP with more structure by definition of files. First, models are created to talk to the database and represent an object. Functions that adhere to each object are written in these files. Then, controllers are files that have routes and direct the server to get information from models or load views. This tells where to go and what information to grab. Finally, views are files that the client sees directly. An example of this is HTML that places things on a webpage. Jinja is a great templating engine to use alongside Python.

Object Relational Mapping(ORM)

ORM is a tequnique that uses OOP to query and manipulate database data. Here we map data to be shown. This can be done by creating data dictionaries. Encapsulation of SQL quieries and hiding them in objects is how this is completed. ORM, essentially, connects an object in memory to a relational database.
